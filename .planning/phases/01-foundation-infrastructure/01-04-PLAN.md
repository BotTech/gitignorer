<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Command handlers → Formatters | Validated result objects (trusted) |
| Formatters → stdout/stderr | Formatted output (data to stdout, messages to stderr) |
| NDJSON stream → process.stdout | Streaming JSON lines (non-blocking) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | formatJSON/formatJSONCompact | mitigate | Only outputs data explicitly passed by commands. Commands control what data is formatted. No sensitive data leaked by formatters themselves. |
| T-04-02 | Denial of Service | streamNDJSON async loop | mitigate | Uses async generator (for await) for non-blocking streaming. Large outputs don't block event loop. |
| T-04-03 | Information Disclosure | formatHuman with @clack/prompts | mitigate | @clack/prompts outputs to stderr (messages), not stdout (data). No sensitive data in styled output in Phase 1 (stubs). |
| T-04-04 | Tampering | NDJSON injection | mitigate | Each item is stringified via JSON.stringify() which escapes control characters. No raw string interpolation in NDJSON output. |
</threat_model>
