<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User terminal → CLI process | Command-line arguments and flags (untrusted user input) |
| CLI process → Node.js runtime | Process exit codes (trusted) |
| Error handler → stderr | Error messages (must not leak sensitive data) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Mode detection | accept | TTY detection is local process state, not security-critical. False positives only affect UX, not security. |
| T-01-02 | Tampering | Exit code mapping | mitigate | Ensure exit codes are constants (1-4), never derived from user input. handleTopLevelError validates error types before mapping. |
| T-01-03 | Information Disclosure | Error messages to stderr | mitigate | Error handler only outputs structured JSON (err.toJSON()), never stack traces or raw user input. Error classes control toJSON() output. |
| T-01-04 | Denial of Service | Blocking I/O in parseAsync | mitigate | Use parseAsync() (non-blocking) throughout. Never use synchronous file operations in async context. |
| T-01-05 | Elevation of Privilege | N/A | accept | No privilege escalation in CLI entry point. Process runs with user's existing permissions. |
</threat_model>
